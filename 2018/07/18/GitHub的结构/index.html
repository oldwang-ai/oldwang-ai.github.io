GitHub的结构

最近要用到opencv扩展模块，就到opencv官网上找，直接点击了网站下方的GitHub链接，然后就看到了opencv扩展模块的repository，即opencv_contrib。点进去翻到最后，发现最晚的版本是3.0.0-beta版，而我现在用的是opencv 2.4.9版本，难道之前的版本没有扩展模块吗？

到处搜索，找答案，后来发现应该是3.0之前的版本，扩展模块没有单独列出。我之前用2.4.9版本时，一般都是在自己的代码工程中设置好依赖的「包含」和「库」的目录，没有太多关注源码，更没有仔细看过sources这个文件夹里面的内容。

其实sources里面就是当前版本的源码，而2.4.9版本的sources目录下有个modules\contrib，这就是2.4.9的扩展模块。

需要自己编译

要使用扩展模块，是需要自己编译的，就是用CMake工具，配好opencv和contrib的路径，然后执行一系列的操作，网上有详细步骤。

不熟悉opencv源码结构

这次之所以花费了很多时间去找2.4.9的扩展模块，是因为：

  1. 不了解opencv源码的结构
  2. 不了解opencv不同版本在GitHub上的演进
GitHub的结构

我想，首先应该熟悉GitHub。当我们登陆 https://github.com/ 后。

在头像左侧有个加号：

  • New repository
  • Import repository

新建仓库和导入仓库，我们知道GitHub是托管代码用的，可以利用这两个选项在GitHub上建立「代码仓库」。

另外两个是：

  • New gist
  • New organization

这两个没用过，暂时不说。

然后点击头像上面的Your profile，可以看到有Overview、Repositories、Stars、Followers、Following几个选项，我之前建了两个代码仓库，所以Repositories这里标记为2。

点击进入博客的Repositories，有下面几个页面：

  • Code
  • Issues
  • Pull requests
  • Projects
  • Wiki
  • Insights
  • Settings
